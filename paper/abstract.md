# Draft Abstract

Large language model serving is often constrained by memory capacity and the cost of moving tensors across memory tiers. We study a learned memory-placement policy that decides whether model tensors should remain on GPU memory or be offloaded to CPU memory under a fixed GPU budget. Our system records tensor-level traces during inference, including size, layer, tensor type, access frequency, phase, and observed latency, and uses those traces to train a lightweight neural policy for GPU vs CPU placement. We compare the learned policy against all-GPU placement, naive overflow, and a hand-written heuristic across multiple memory budgets. The starter repository in this project focuses on building the tracing, simulation, training, and benchmarking backbone needed to support that evaluation.
